# Тиждень 1: Проектування та фундамент
- Ініціалізація проекту: Створення ASP.NET Core Razor Pages проекту (Web App).

- Налаштування Бази Даних: Підключення Entity Framework Core та конфігурація LocalDB у файлі appsettings.json.

- Створення моделей: Розробка базових класів Book, Review та Feedback з атрибутами валідації ([Required], [StringLength]).

# Тиждень 2: Система авторизації (Identity)
- Інтеграція Identity: Додавання стандартної системи автентифікації Microsoft Identity.

- Об'єднання контекстів: Злиття контекстів бази даних (Identity та Книг) в один спільний RazorPagesBookContext для цілісності даних.

- Ролі та доступ: Створення ролі "Admin" та обмеження доступу до папки /Admin.

- Seed Data: Написання скрипту для автоматичного створення тестових книг та профілю адміністратора при першому запуску.

# Тиждень 3: Функціонал для користувачів
- Каталог книг: Створення сторінок для перегляду списку всіх книг та детальної інформації про кожну.

- Система відгуків: Розробка форми під кожною книгою для додавання коментарів та виставлення рейтингу (зірочок).

- Головна сторінка: Динамічний висновок "Новинок" та "Останніх відгуків" замість статичного вітання.

# Тиждень 4: Адмін-панель та зворотний зв'язок
- Панель керування (CRUD): Створення сторінок для адміна, де можна додавати нові книги та редагувати існуючі.

- Адмін-відповідь: Реалізація логіки відповідей адміністратора на відгуки користувачів.

- Email-сервіс: Створення інтерфейсу IEmailSender та "заглушки" (EmailStub) для імітації відправки листів-підтверджень.

# Тиждень 5: Полірування та публікація
- Дизайн (UI/UX): Адаптація інтерфейсу за допомогою Bootstrap, додавання іконок (Bootstrap Icons).

- Рефакторинг: Виправлення помилок з типами даних (перетворення Rating зі string в int).

- Документація: Написання README.md, створення архітектурних діаграм та описів сценаріїв.

- Деплой: Завантаження коду на GitHub та фінальне тестування міграцій.
