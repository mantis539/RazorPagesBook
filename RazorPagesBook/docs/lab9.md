Ця схема демонструє повний цикл CI/CD та розгортання сервісу в Kubernetes. Процес починається з git push коду в репозиторій GitHub,
що автоматично запускає пайплайн GitHub Actions для збірки Docker-образу та його відправки в Container Registry. 
Усередині кластера об’єкт Deployment завантажує цей образ і створює групу Pod-ів у виділеному просторі імен (Namespace). 
Зовнішній користувач отримує доступ до застосунку через DNS та Ingress, який маршрутизує вхідний трафік до внутрішнього Service, 
що рівномірно розподіляє запити між працюючими подами.
